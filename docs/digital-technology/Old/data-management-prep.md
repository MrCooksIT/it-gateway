# Data and Information — Grade 8 & 9 Revision

## Key concepts to know

- The difference between **data** and **information**
- How **spreadsheets** organise and process data
- Basic **database** concepts (tables, records, fields)
- What **data validation** means
- File types and data storage formats
- Units of digital storage (bit, byte, KB, MB, GB, TB)

## Quick definitions

| Term | Meaning |
|---|---|
| Data | Raw, unprocessed facts and figures (e.g. 42, "Cape Town") |
| Information | Data that has been processed to give it meaning |
| Database | An organised collection of related data |
| Table | A structure in a database that stores data in rows and columns |
| Record | One row in a database table — represents one item |
| Field | One column in a database table — stores one type of data |
| Validation | Checking that data entered meets specific rules |
| Byte | Basic unit of digital storage (8 bits) |

## Storage units table

| Unit | Abbreviation | Size |
|---|---|---|
| Kilobyte | KB | 1 024 bytes |
| Megabyte | MB | 1 024 KB |
| Gigabyte | GB | 1 024 MB |
| Terabyte | TB | 1 024 GB |

## Practice questions

1. Explain the difference between data and information. Use an example.
2. A school stores learner details in a spreadsheet with columns: Name, Surname, Grade, Age. 
   - What is each column called in a database?
   - What is each row called in a database?
3. A form only allows ages between 5 and 18 to be entered. What is this an example of?
4. Convert: How many KB are in 2 MB?
5. Give **one** example each of data that is a number, text, and a date.
6. Why is it important to validate data before storing it in a database?
7. A file is 500 MB. Is it likely to be a text document, a music file, or a short video? Explain your answer.

## Typical test format

Your test on this topic will likely include:
- Data vs information scenarios
- Storage unit conversions
- Database vocabulary (field, record, table, key)
- Validation rules and why they matter
