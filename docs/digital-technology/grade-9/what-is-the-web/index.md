# Unit Overview: What is the Web?

## Introduction

Before you can build websites or understand how digital technology shapes the modern world, you need to understand the foundation everything runs on: **the internet and the World Wide Web**. This unit takes you on a journey from the physical cables and satellites that carry data around the globe, all the way to the moment a web page appears on your screen.

This is not just background knowledge. Understanding how the web works makes you a smarter user, a safer digital citizen, and a better web developer. When something goes wrong online, you will know why. When you design a website, you will understand what is happening beneath the surface.

## Learning Outcomes

By the end of this unit, you should be able to:

### Knowledge and Understanding
- Explain the difference between the internet and the World Wide Web
- Describe how data travels across the internet using packets, routers, and the TCP/IP protocol
- Explain what an IP address is and distinguish between IPv4 and IPv6
- Identify the major types of internet infrastructure (servers, data centres, undersea cables, satellites)
- Explain how internet service providers (ISPs) connect users to the internet
- Describe the role of South Africa's internet landscape, including challenges and mobile dominance
- Explain how a web browser retrieves and displays a web page
- Compare the major web browsers and their rendering engines
- Break down a URL into its component parts and explain what each part means
- Explain how DNS (Domain Name System) translates domain names into IP addresses
- Distinguish between HTTP and HTTPS, and explain why HTTPS matters
- Interpret common HTTP status codes (200, 404, 403, 500)
- Identify different types of websites and the roles involved in building them
- Apply good web design principles when planning or evaluating a website

### Skills
- Use browser developer tools to inspect a web page's source code
- Identify whether a website is secure (HTTPS) using browser indicators
- Evaluate a website's design against established design principles
- Recognise signs of poor or unsafe web design

### Values and Attitudes
- Appreciate the scale and complexity of the infrastructure behind everyday internet use
- Understand the importance of internet access as a tool for equality and opportunity in South Africa
- Take a critical approach to online privacy and browser safety

## Why Understanding the Web Matters

Think about how much of your daily life involves the internet: social media, streaming music and video, online banking, school research, messaging friends. South Africa has over 45 million internet users, and that number grows every year. The internet is not just entertainment — it is increasingly essential for education, employment, healthcare, and civic participation.

Yet most people who use the internet every day have very little idea how it actually works. They treat it like magic: type something in, get something back. This unit is about pulling back the curtain.

:::info Why this matters for web development
If you want to build websites — which is what the rest of this course is about — you absolutely need to understand the environment your websites live in. Knowing about HTTP helps you understand why pages sometimes fail to load. Knowing about DNS helps you understand how domain names work. Knowing about browsers helps you understand why the same page can look different in Chrome vs Safari.
:::

When you finish this unit, you will not just be a user of the web. You will understand it.

## Key Vocabulary

Before you start, here are some key terms you will encounter throughout this unit:

| Term | Meaning |
|------|---------|
| Internet | The global network of interconnected computers |
| World Wide Web (WWW) | A system of web pages and links that runs on the internet |
| Browser | Software used to access and display web pages |
| URL | Uniform Resource Locator — the address of a web page |
| DNS | Domain Name System — translates domain names to IP addresses |
| HTTP/HTTPS | Protocols used to send web pages between servers and browsers |
| IP Address | A unique number that identifies a device on the internet |
| Server | A computer that stores and serves web pages to users |
| ISP | Internet Service Provider — a company that provides internet access |
| Bandwidth | The amount of data that can be transmitted per second |
