# Web Design

## What is Web Design?

**Web design** is the process of planning and creating the appearance, layout, structure, and user experience of websites. A web designer decides how a website looks: the colours, fonts, images, spacing, navigation, and the overall experience a visitor has when they use the site.

Web design is both a technical skill and a creative art. The best websites are not just visually attractive — they are also easy to use, fast, accessible to everyone, and effective at achieving their purpose.

:::tip Key Term
**Web Design:** The process of planning and creating the visual appearance, layout, and user experience of a website, combining principles from graphic design, communication, and technology.
:::

---

## Types of Websites

Websites come in many different forms, each with a distinct purpose:

| Type | Purpose | Examples |
|------|---------|---------|
| **Informational** | Provide information about a person, company, or topic | Company websites, Wikipedia |
| **E-commerce** | Sell products or services online | Takealot, Amazon, Woolworths Food |
| **Social Media** | Connect people and share content | Instagram, TikTok, X (Twitter), Facebook |
| **Blog / News** | Share articles, opinions, or news | IOL News, Daily Maverick, personal blogs |
| **Portfolio** | Showcase creative work | An artist's or developer's personal site |
| **Government Services** | Provide official information and online services | SARS eFiling, Home Affairs |
| **Educational** | Provide learning content | Khan Academy, Coursera, CodeHS |
| **Entertainment** | Provide media and entertainment | Netflix, YouTube, Spotify |
| **Non-profit** | Raise awareness or funds for a cause | Organisation websites |

Each type of website has different design needs. An e-commerce site needs a clear shopping cart and checkout process. A portfolio site needs large, high-quality images. A government site needs to be simple and accessible to all citizens.

## Roles in Web Development

Building a website involves many different types of work. In a professional setting, these are often done by different people:

### Web Designer
Responsible for the visual appearance of the website. They decide on colour schemes, typography, layout, and visual style. Web designers typically use tools like **Adobe Photoshop**, **Figma**, or **Adobe XD** to create mockups and prototypes before any code is written.

### Web Developer (Front-End)
Turns the designer's visual mockup into a working website using **HTML**, **CSS**, and **JavaScript**. A front-end developer builds everything the user sees and interacts with.

### Web Developer (Back-End)
Works on the server side — databases, user authentication, data processing. When you log into a website or place an order, back-end code is doing that work. Back-end developers use languages like Python, PHP, Node.js, and databases like MySQL.

### Full-Stack Developer
A developer who can work on both the front-end and back-end — "the full stack."

### UX Designer (User Experience Designer)
Focuses on how easy and enjoyable the website is to use. UX designers research users, create user journeys, design wireframes, and test prototypes with real users. Their goal is to make sure the website serves the user's needs effectively.

### Content Creator / Copywriter
Writes the text (called **copy**) for the website and creates or curates images, videos, and other media.

### SEO Specialist
Optimises the website so it ranks highly in search engine results (Google, Bing). SEO stands for Search Engine Optimisation.

:::info Small Projects vs Large Teams
For a small personal website, one person might handle all of these roles. For a large commercial website — say, a major bank's internet banking platform — you might have hundreds of people in specialised roles working simultaneously.
:::

## Principles of Good Web Design

Great web design is not accidental. There are well-established principles that guide the creation of effective, enjoyable websites.

### 1. Simplicity and Clarity
A good website is easy to understand at a glance. Visitors should immediately understand what the site is about and how to find what they need. **Remove everything that is not necessary.** Cluttered, busy designs confuse users and drive them away.

### 2. Consistent Navigation
The navigation menu should be in the same place on every page of the site. Users should always be able to answer three questions: Where am I? Where have I been? Where can I go? If navigation is inconsistent or confusing, users will leave.

### 3. Readable Typography
Text must be easy to read. This means:
- Font size should be large enough (at least 16px for body text)
- Good contrast between text and background (dark text on light background, or vice versa)
- Not too many different fonts — usually one for headings and one for body text is enough
- Sufficient line spacing (space between lines of text)

### 4. Good Use of White Space
**White space** (also called negative space) is the empty space around and between elements on a page. It is not wasted space — it is breathing room that makes a design feel clean, modern, and easy to read. Cramming too much onto a page creates visual stress.

### 5. Mobile Responsiveness
With the majority of South African internet users browsing on mobile phones, a website **must** work well on small screens. Responsive design means the layout adapts automatically to different screen sizes. A site that looks great on a desktop but breaks on a phone is failing more than half its potential users.

### 6. Fast Loading
Studies show that users leave a website if it takes more than **3 seconds** to load. Slow sites lose visitors, rank lower in Google, and frustrate users. Good design includes technical optimisation: small image file sizes, clean code, and efficient hosting.

### 7. Accessibility
Websites should be usable by everyone, including people with visual, hearing, motor, or cognitive disabilities. This includes:
- Using alt text on images (for screen readers)
- Good colour contrast (for low-vision users)
- Keyboard navigation (for users who cannot use a mouse)
- Clear, simple language

:::warning Accessibility is Not Optional
In many countries, websites — especially government and public services — are legally required to meet accessibility standards. Beyond the legal requirement, it is simply the right thing to do: an inaccessible website excludes a significant portion of the population.
:::

### 8. Clear Calls to Action
A **call to action (CTA)** is something you want the user to do: "Buy Now", "Sign Up", "Contact Us". Good design makes CTAs obvious and easy to find.

## Colour Theory Basics

Colour is one of the most powerful tools in a designer's toolkit. It creates mood, builds brand identity, and guides user attention.

### The Colour Wheel
The colour wheel shows the relationships between colours:
- **Primary colours:** Red, Blue, Yellow
- **Secondary colours:** Orange, Green, Purple (made by mixing two primaries)
- **Tertiary colours:** Mixes of primary and secondary colours

### Complementary Colours
**Complementary colours** sit opposite each other on the colour wheel (like red and green, or blue and orange). Using complementary colours together creates strong visual contrast and can make elements stand out dramatically.

### Colour Psychology
Different colours evoke different emotions and associations:

| Colour | Common Associations | Used By |
|--------|--------------------|---------| 
| **Red** | Energy, urgency, passion, danger | Coca-Cola, YouTube, Netflix |
| **Blue** | Trust, calm, professionalism | Facebook, Samsung, banks |
| **Green** | Nature, health, growth, money | Whole Foods, WhatsApp |
| **Yellow** | Optimism, warmth, caution | McDonald's, Snapchat |
| **Orange** | Creativity, enthusiasm, affordable | Amazon, Harley-Davidson |
| **Purple** | Luxury, creativity, wisdom | Cadbury, Hallmark |
| **Black** | Sophistication, power, elegance | Apple, Chanel, Nike |

:::info Colour in South African Context
Be mindful that colour associations can differ across cultures. White, for example, is associated with purity in Western cultures but with mourning in some Eastern cultures. When designing for a specific audience, research the cultural context of your colour choices.
:::

## Typography

**Typography** is the art and technique of arranging text. The fonts you choose and how you use them significantly affect how professional, readable, and on-brand a website feels.

### Choosing Readable Fonts
- **Serif fonts** (like Times New Roman, Georgia) have small decorative strokes on the ends of letters. They are traditional and associated with print newspapers, books, and formal documents.
- **Sans-serif fonts** (like Arial, Helvetica, Roboto) have clean, simple strokes with no decorations. They are modern and generally easier to read on screens.
- **Monospace fonts** (like Courier New, Consolas) give every character the same width. Used for code.

### Font Pairing
Using two complementary fonts — one for headings, one for body text — creates visual hierarchy while maintaining variety. A common pairing is a bold sans-serif for headings and a readable serif or sans-serif for body text.

### Typographic Hierarchy
Size, weight, and colour guide the reader's eye to the most important information first. A clear hierarchy looks like this:
- Main heading: **large, bold**
- Sub-heading: **medium, semi-bold**
- Body text: regular weight, comfortable reading size
- Caption: small, lighter colour

## Layout: Organising the Page

Layout is about deciding where things go on a page and why.

### Grid Systems
Professional web designers use **grid systems** — a set of invisible columns and rows — to create consistent, orderly layouts. Most websites use 12-column grids because 12 is divisible by 2, 3, 4, and 6, giving maximum flexibility.

### The F-Pattern
Research using eye-tracking technology has shown that people reading web pages in English tend to scan in an **F-pattern**:
1. First, a horizontal scan across the top
2. Then, a second horizontal scan a bit lower
3. Then, a vertical scan down the left side

This means the most important content should be placed along the top and left side of the page.

### Above the Fold
The phrase **"above the fold"** comes from newspaper design — the top half of a folded newspaper is what people see first on a newsstand. On the web, it refers to the part of the page visible without scrolling. Important information and calls to action should be above the fold.

## User Experience (UX)

**User Experience (UX)** is how a person feels when they interact with a website or application. A site can look beautiful but still have terrible UX if it is confusing, slow, or frustrating to use.

:::tip Key Term
**UX (User Experience):** The overall experience a user has when interacting with a website or application — how easy, enjoyable, and effective it is to use.
:::

### Key UX Questions
- Can a first-time visitor find what they are looking for within three clicks?
- Is the language clear and appropriate for the target audience?
- Are error messages helpful and friendly?
- Does the site work consistently across different devices and browsers?
- Is feedback given when the user takes an action (e.g., a confirmation message after submitting a form)?

## The Three Core Technologies

Every website is built from a combination of three technologies, each with a distinct role:

| Technology | Role | Analogy |
|-----------|------|---------|
| **HTML** | Structure — defines what content exists and what type it is | The skeleton or framework of a building |
| **CSS** | Style — controls colours, fonts, layout, and appearance | The paint, decor, and furniture |
| **JavaScript** | Behaviour — adds interactivity and dynamic features | The electricity and plumbing |

:::info Separation of Concerns
Keeping these three technologies separate is a fundamental principle of good web development. HTML defines the content, CSS defines how it looks, and JavaScript defines how it behaves. Mixing them together (e.g., writing style rules directly in HTML) makes code messy and hard to maintain.
:::

## Website Builders vs Writing Code

You do not have to know how to code to create a website. **Website builders** like Wix, Squarespace, and Weebly allow anyone to create a website using drag-and-drop tools with no coding required.

| | Website Builders | Writing Code |
|--|----------------|-------------|
| **Skill required** | None — drag and drop | HTML, CSS, JavaScript |
| **Time to build** | Hours | Days to weeks |
| **Flexibility** | Limited by templates | Unlimited |
| **Cost** | Monthly subscription | Hosting only |
| **Performance** | Often slower (bloated code) | Can be highly optimised |
| **Learning value** | Very little | Very high |
| **Career value** | None | High — employable skill |

Website builders are useful for simple personal or small business sites, especially when you need something quickly. But if you want a career in technology or want to build exactly what you imagine, you need to learn to code.

## Responsive Web Design

**Responsive web design** means building a website that automatically adapts its layout to suit the screen size of the device being used. The same website will look appropriate on a wide desktop monitor, a tablet, and a small smartphone screen.

:::tip Key Term
**Responsive Web Design:** An approach to web design where the layout and appearance of a website automatically adjusts to fit different screen sizes and orientations.
:::

This is achieved primarily through **CSS media queries** — rules that apply different styles depending on the screen width. For example:
- On a wide desktop: a three-column layout
- On a tablet: a two-column layout
- On a phone: a single-column layout

Since most South Africans browse the web on mobile phones, responsive design is not optional — it is essential.
