# Algorithms

Before writing a single line of code, a good programmer designs a solution. Algorithms are step-by-step instructions for solving a problem — they can be expressed as pseudocode, flowcharts or structured English. Mastering algorithm design makes everything else in programming easier.

## Pseudocode vs Flowcharts

South African IT examinations may ask you to write **pseudocode** or draw/interpret a **flowchart**. Both represent the same logic in different ways:

| Tool | Strengths |
|---|---|
| Pseudocode | Quick to write, easy to convert to code, language-independent |
| Flowchart | Visually clear, great for showing branches and loops |

## CAPS alignment

Algorithms appear in both **Paper 1** (design and implementation) and **Paper 2** (tracing and explaining) across Grades 10–12. In Paper 1, algorithm design is usually the first question and sets up the Delphi coding that follows.
