# Data & Information Management

This topic covers the theory behind how data is stored, organised and managed using databases. Students learn how to design a well-structured relational database, how to apply normalisation rules, and how to ensure data quality through integrity constraints. The practical application of SQL is covered in the Practical section; this section focuses on the underlying concepts.

## CAPS alignment

Data & Information Management is assessed in **Paper 2** across all grades.

- **Grade 10** — Data types, flat-file vs relational concepts, basic database terminology.
- **Grade 11** — ER diagrams, primary and foreign keys, introductory normalisation.
- **Grade 12** — Full normalisation to 3NF, data integrity, SQL theory, DBMS roles and features.
