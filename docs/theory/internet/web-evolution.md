# Web Evolution

The web you use today is nothing like the web of the 1990s. From static pages that only showed text to interactive platforms where you create content, to AI-driven experiences — the web has evolved through three distinct generations.

> [!NOTE] Grade 11
> Web evolution (Web 1.0, 2.0, 3.0) is primarily a Grade 11 topic. Grade 12 extends this with semantic web, scripting languages, and emerging technologies.

---

## The Three Generations of the Web

| | **Web 1.0** | **Web 2.0** | **Web 3.0** |
|---|---|---|---|
| **Era** | ~1991–2004 | ~2004–present | Emerging |
| **Type** | Read-only | Read-Write | Semantic / Intelligent |
| **Content created by** | Website owners only | Users and owners | Machines and users |
| **User role** | Passive consumer | Active participant | Intelligent interaction |
| **Key technologies** | Static HTML, GIF images | AJAX, APIs, JavaScript, CSS | AI, blockchain, RDF, linked data |
| **Interaction** | None — pages are fixed | Comments, likes, sharing, uploads | Personalised, context-aware |
| **Examples** | Early news sites, brochure pages | Facebook, YouTube, Wikipedia, Twitter | AI assistants, IoT dashboards, smart search |

---

## Web 1.0 — The Read-Only Web

The first generation of the web (early-to-mid 1990s through early 2000s).

**Characteristics:**
- Static HTML pages — content never changes unless the webmaster edits the file
- No user interaction — you read but cannot contribute
- No user accounts, no comments, no dynamic content
- Pages linked together with hyperlinks
- Limited multimedia (mostly text and images)
- Dial-up internet — slow loading

**Technology:**
- Static HTML files on web servers
- Basic CSS for styling
- No databases behind the pages

**Examples:**
- Brochure-style company websites
- Early news archives
- Personal "homepage" sites
- Government information pages

---

## Web 2.0 — The Read-Write Web

The second generation, emerging around 2004–2005 and continuing today.

**Characteristics:**
- Users **create, share, and interact** with content
- User-generated content is central
- Social networking, blogs, wikis, video hosting
- Web applications that function like desktop software
- Personalisation — your experience differs from someone else's

**Key technologies:**
| Technology | Role |
|---|---|
| **AJAX** | Updates parts of a page without full reload (Asynchronous JavaScript and XML) |
| **JavaScript** | Adds interactivity and dynamic behaviour to pages |
| **CSS frameworks** | Responsive design, consistent styling |
| **APIs** | Applications communicate and share data with each other |
| **Databases** | Dynamic content served from server-side databases |
| **Rich multimedia** | Embedded video, audio, interactive maps |

**Examples:**
- **Social networks**: Facebook, Instagram, X (Twitter)
- **Video platforms**: YouTube, TikTok
- **Collaborative tools**: Wikipedia, Google Docs
- **Blogging**: WordPress, Blogger
- **Shopping**: Amazon (reviews, recommendations)
- **Email as web app**: Gmail (no page reload)

**Web 2.0 risks:**
- Privacy — users share vast personal data
- Misinformation — anyone can publish anything
- Addiction by design — engagement algorithms
- Data ownership — platform owns your content

---

## Web 3.0 — The Semantic Web

The third generation is still emerging, with different definitions in use:

### Semantic Web (Tim Berners-Lee's vision)
The web of **meaningful data** — machines can understand content, not just display it.

- Current web: pages are for humans to read
- Semantic web: data tagged with meaning so machines can process it
- Uses **RDF** (Resource Description Framework) and **ontologies** to describe relationships

**Example:** Instead of searching "hotels near Cape Town", a semantic web agent finds available rooms, compares prices, and books for you — understanding context and meaning.

### Web 3.0 / Decentralised Web
Also used to mean a **decentralised web** built on blockchain:
- No central platform controls your data
- Users own their own digital identity and assets
- **NFTs**, **cryptocurrency**, **decentralised apps (dApps)**

### AI-Driven Web
Web 3.0 also describes AI-enhanced experiences:
- AI assistants understand natural language queries
- Personalised content recommendations
- Voice search and conversational interfaces
- Context-aware responses

**Examples of Web 3.0 concepts:**
- AI chat assistants (ChatGPT, Gemini)
- Smart search understanding intent, not just keywords
- IoT dashboards connecting physical and digital worlds
- Decentralised social media platforms

## Web Evolution Summary

```
Web 1.0: "Here is information" (one-way broadcast)
Web 2.0: "Let's interact and share" (two-way participation)
Web 3.0: "I understand what you mean" (intelligent interpretation)
```

## Static vs Dynamic Websites

| Feature | Static Website | Dynamic Website |
|---|---|---|
| Content | Fixed HTML — same for everyone | Generated per request — personalised |
| Database | No | Yes (SQL/NoSQL database) |
| User interaction | Minimal | Extensive (login, forms, transactions) |
| Languages | HTML + CSS only | HTML + CSS + PHP/Python + Database |
| Update method | Edit HTML files manually | Update database → all pages reflect change |
| Speed | Fast (no processing) | Slightly slower (server processing) |
| Cost | Low | Higher |
| Examples | Portfolio site, info page | Online banking, social media, e-commerce |

## Intranet vs Extranet vs Internet

| | **Internet** | **Intranet** | **Extranet** |
|---|---|---|---|
| Access | Anyone | Internal staff only | Authorised external parties |
| Ownership | Public infrastructure | Organisation's private network | Organisation's private network |
| Content | Everything | Internal documents, tools | Specific shared resources |
| Examples | WWW | Staff portal, internal email | Supplier ordering system, patient portal |

## Emerging Web Technologies

| Technology | Description |
|---|---|
| **Progressive Web Apps (PWA)** | Web apps that work like native mobile apps (offline, push notifications) |
| **WebAssembly** | Near-native performance code running in the browser |
| **Voice search** | Natural language queries via voice assistants |
| **Augmented Reality (AR) web** | AR experiences in browsers (WebXR) |
| **AI integration** | AI APIs powering web services |

## Key Terms

| Term | Definition |
|---|---|
| **Web 1.0** | First-generation read-only web with static pages |
| **Web 2.0** | Second-generation interactive web with user-generated content |
| **Web 3.0** | Third-generation semantic/AI-driven/decentralised web |
| **Static website** | Website with fixed HTML content — same for every visitor |
| **Dynamic website** | Website that generates content from a database per request |
| **AJAX** | Technology for updating page content without full reload |
| **API** | Application Programming Interface — allows applications to communicate |
| **Semantic web** | Vision of web where data has machine-readable meaning |
| **Intranet** | Private network accessible only to internal staff |
| **Extranet** | Private network extended to authorised external users |

## Exam Focus

> [!IMPORTANT] High-Frequency Questions
>
> 1. **"What is the main difference between Web 1.0 and Web 2.0?"** — Web 1.0 is read-only; content created only by website owners. Web 2.0 is read-write; users create and share content (social media, YouTube, Wikipedia)
>
> 2. **"Give TWO characteristics of Web 2.0"** — user-generated content; social networking; interactive applications; personalised experience; AJAX updates without page reload
>
> 3. **"What is the semantic web (Web 3.0)?"** — A web where data has machine-readable meaning; machines can understand context and process information intelligently
>
> 4. **"Explain the difference between a static and dynamic website"** — Static: fixed HTML, same for all visitors, no database; Dynamic: content generated from database per request, personalised per user
>
> 5. **"What is the difference between an intranet and an extranet?"** — Intranet: accessible to internal employees only; Extranet: extends access to authorised external parties (suppliers, clients)
