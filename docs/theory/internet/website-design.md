# Website Design

A website that looks beautiful but confuses visitors has failed at its job. Good web design is about more than aesthetics — it's about making sure users can find what they need quickly, on any device, in any condition.

> [!NOTE] Grade 10–11
> Website design principles and evaluation are Grade 10–11 content. Grade 12 extends with scripting, backend technologies, and web standards.

---

## Principles of Good Website Design

### 1. Layout and Structure
- **Clear visual hierarchy** — headings, subheadings, body text at distinct sizes
- **Consistent layout** — same navigation position on every page
- **White space** — breathing room between elements reduces cognitive load
- **Grid system** — content aligned to a consistent grid

### 2. Navigation
- **Intuitive navigation** — users should find what they need without thinking
- **Persistent navigation bar** — always visible, same position
- **Breadcrumbs** — show current position (Home > Theory > Networks > Topologies)
- **Search function** — for large sites
- **3-click rule** — users should reach any page in 3 clicks or fewer

### 3. Content
- **Relevant and accurate** — current, correct information
- **Concise** — web users scan, not read — short paragraphs, bullet points
- **Readable** — sufficient font size (min 16px), high contrast
- **Clear headings** — users scan headings to find relevant sections
- **No broken links** — regular maintenance required

### 4. Visual Design
- **Consistent colour scheme** — limited palette (3–4 colours)
- **Font consistency** — max 2–3 font families
- **High contrast** — text must contrast clearly with background (WCAG standard: 4.5:1 ratio)
- **Quality images** — optimised (compressed) for fast loading
- **Brand consistency** — colours, fonts, tone match the organisation

### 5. Performance
- **Fast loading** — compress images, minimise CSS/JS files
- **Minimal HTTP requests** — combine files where possible
- **Content Delivery Network (CDN)** — serve static files from servers near the user

### 6. Accessibility
- **Alt text on all images** — for screen readers and when images fail to load
- **Keyboard navigation** — all functionality accessible without a mouse
- **Captions on videos** — for deaf/hard-of-hearing users
- **Sufficient colour contrast** — for colour-blind users
- **Responsive design** — works on all screen sizes

### 7. Responsiveness
- **Responsive design** — layout adapts to screen size (desktop, tablet, phone)
- Uses CSS media queries to adjust layout
- Images scale proportionally
- Navigation collapses to "hamburger menu" on small screens

## Website Evaluation Criteria (ACCPO)

When evaluating a website's credibility and quality:

| Criterion | Questions to ask |
|---|---|
| **Authority** | Who created this? Are they qualified? Is there an "About" page? Is the author named? |
| **Accuracy** | Is the information correct? Are sources cited? Can you verify facts elsewhere? |
| **Currency** | When was it last updated? Is the information still relevant? Are dates shown? |
| **Purpose** | Why was it created? To inform, sell, persuade, or entertain? Is there bias? |
| **Objectivity** | Is information presented fairly? Are multiple viewpoints shown? Any obvious bias? |

> [!TIP] Quick evaluation check
> Check the URL: `.gov` = government; `.ac` or `.edu` = education; `.org` = organisation — generally more credible than `.com` for factual information

## W3C — World Wide Web Consortium

**W3C** is the international standards organisation for the web, founded by Tim Berners-Lee.

**What W3C does:**
- Sets standards for HTML, CSS, accessibility, and web technologies
- Publishes guidelines including:
  - **WCAG** (Web Content Accessibility Guidelines)
  - **HTML5** and **CSS3** specifications
  - Web semantics standards

**Why follow W3C standards:**
- Ensures cross-browser compatibility
- Improves accessibility
- Better SEO (search engines reward standards-compliant sites)

## Web Design Technologies

| Technology | Role |
|---|---|
| **HTML** | Structure and content — headings, paragraphs, links, images |
| **CSS** | Presentation — colours, fonts, layout, spacing |
| **JavaScript** | Interactivity — button clicks, form validation, dynamic content |
| **PHP** | Server-side scripting — generates dynamic content from database |
| **MySQL / SQL** | Database storing dynamic content |
| **CMS** | Content Management System — allows non-technical editing (WordPress, Joomla) |

## User Experience (UX) vs User Interface (UI)

| | User Interface (UI) | User Experience (UX) |
|---|---|---|
| What | Visual design — what it looks like | Overall experience — how it feels to use |
| Focus | Aesthetics, layout, colour, typography | Ease of use, user journey, satisfaction |
| Example | Button colour and size | Whether the button was easy to find and use |

Good design needs both: beautiful (UI) AND functional (UX).

## Common Website Design Mistakes

| Mistake | Problem |
|---|---|
| Too many colours/fonts | Looks unprofessional, confusing |
| Slow loading images | Users leave if page takes > 3 seconds |
| No mobile optimisation | ~60% of web traffic is mobile |
| Broken links | Frustrates users; harms SEO |
| Too much text on one page | Users don't read walls of text |
| Auto-playing audio/video | Annoying — users close immediately |
| No search function (large sites) | Users can't find content |
| Inaccessible content | Excludes users with disabilities |

## Exam Focus

> [!IMPORTANT] High-Frequency Questions
>
> 1. **"Give FOUR characteristics of a well-designed website"** — consistent navigation; fast loading; responsive (mobile-friendly); high contrast text; accurate and current content; accessibility
>
> 2. **"Explain the ACCPO criteria for evaluating a website"** — Authority (who created it?); Accuracy (is it correct?); Currency (when updated?); Purpose (why created?); Objectivity (is it biased?)
>
> 3. **"What is responsive web design?"** — Design that automatically adapts the layout to different screen sizes (desktop, tablet, phone) so the site is usable on any device
>
> 4. **"What is the role of W3C?"** — International organisation that sets web standards (HTML, CSS, accessibility guidelines) to ensure compatibility and accessibility across browsers
>
> 5. **"Give TWO accessibility features a website should include"** — alt text on images; captions on videos; keyboard navigation; sufficient colour contrast; screen reader compatibility
