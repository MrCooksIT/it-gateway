# E-Communication

Electronic communication has replaced letters, fax machines, and phone trees. Email, instant messaging, video conferencing, and social media all rely on the same underlying networks — but they come with unique protocols, etiquette requirements, and threats.

> [!NOTE] Grade 10–11
> E-communication is introduced at Grade 10 (email basics, netiquette, threats) and extended at Grade 11 (VoIP, RSS, protocols).

## Forms of Electronic Communication

| Type | Description | Examples |
|---|---|---|
| **Email** | Asynchronous text/file communication via mail servers | Gmail, Outlook |
| **Instant messaging (IM)** | Real-time text chat | WhatsApp, Telegram, Teams |
| **VoIP** | Voice (and video) calls over the internet | Zoom, Teams, WhatsApp calls, Skype |
| **Social media** | Sharing content, updates, and communication | Facebook, Instagram, X (Twitter) |
| **RSS feeds** | Subscribing to website updates | News aggregators, podcasts |
| **Discussion forums** | Asynchronous threaded discussions | Reddit, Stack Overflow |
| **Blog / vlog** | Publishing written or video content | WordPress, YouTube |
| **Video conferencing** | Real-time multi-party video calls | Zoom, Google Meet, Teams |

## Email — How it Works

Email relies on three protocols:

| Protocol | Full Name | Purpose |
|---|---|---|
| **SMTP** | Simple Mail Transfer Protocol | **Sending** email (port 25/587) |
| **POP3** | Post Office Protocol v3 | **Downloading** email from server (port 110) — deletes from server |
| **IMAP** | Internet Message Access Protocol | **Accessing** email on server (port 143) — stays on server |

### SMTP Process (sending):
1. You compose and send an email in your email client
2. Client connects to your **outgoing mail server (SMTP)**
3. SMTP server looks up the recipient domain's **MX record** via DNS
4. Connects to the recipient's mail server and delivers the message
5. Message sits in the recipient's mailbox until they download/view it

### POP3 vs IMAP

| Feature | POP3 | IMAP |
|---|---|---|
| Email stored | Downloaded to device | Stays on server |
| Multiple devices | Poor — one device at a time | Excellent — syncs across all devices |
| Offline access | Yes (after download) | Limited (needs connection to view) |
| Storage | Local device | Server storage |
| Best for | Single device | Multiple devices |

## Email Best Practices

### Structure of a professional email:
- **To** — primary recipient(s)
- **CC** (Carbon Copy) — receives a copy; all recipients can see this
- **BCC** (Blind Carbon Copy) — receives a copy; other recipients cannot see BCC list
- **Subject** — clear, specific summary of the email topic
- **Body** — professional, clear language
- **Signature** — name, position, contact details

### Attachment guidelines:
- Keep file sizes small — compress large attachments
- Warn recipient before sending large files
- Never open unexpected attachments from unknown senders

## Netiquette

**Netiquette** (internet etiquette) — rules for respectful and appropriate online communication.

| Rule | Explanation |
|---|---|
| **No caps lock** | TYPING IN ALL CAPS = shouting — considered rude |
| **Use clear subject lines** | Helps recipient prioritise and find email later |
| **Reply promptly** | Respond within 24–48 hours for professional email |
| **Don't forward without permission** | Private emails should not be forwarded |
| **Proofread before sending** | Check spelling, tone, and correct recipients |
| **Don't CC unnecessarily** | Only CC people who need to be informed |
| **Respect others' time** | Be concise — don't send unnecessarily long emails |
| **No spam** | Don't send unsolicited bulk email |
| **Cite sources** | If sharing information, credit the original author |
| **Be respectful** | Online anonymity doesn't excuse rude behaviour |

## VoIP — Voice over Internet Protocol

**VoIP** converts voice (and video) into digital data packets and transmits them over the internet.

| Feature | Detail |
|---|---|
| How it works | Audio → digitised → compressed → sent as data packets → reassembled |
| Advantage | Cheap or free calls anywhere in the world |
| Disadvantage | Quality depends on internet connection; latency can cause echo |
| Examples | Zoom, Microsoft Teams, WhatsApp calls, Skype, Google Meet |

**VoIP vs Traditional phone:**
| | VoIP | Traditional |
|---|---|---|
| Cost | Low (uses existing internet) | Higher (phone network charges) |
| International calls | Cheap/free | Expensive |
| Quality | Varies with internet | Consistent |
| Infrastructure needed | Internet connection | Phone line |

## RSS — Really Simple Syndication

**RSS** is a format that allows users to subscribe to updates from websites and have new content delivered automatically.

- Website publishes an **RSS feed** (XML file) that updates when new content is added
- User subscribes in an **RSS reader** (aggregator)
- New articles, podcasts, or news items appear in the reader automatically
- No need to visit each website individually

**Examples:** news aggregators, podcast subscriptions, blog updates

## Email Threats

| Threat | Description |
|---|---|
| **Spam** | Unsolicited bulk email — usually advertising or malicious |
| **Phishing** | Fake email pretending to be a trusted organisation to steal credentials |
| **Spear phishing** | Targeted phishing aimed at a specific person (more convincing) |
| **Vishing** | Voice phishing — scam phone calls |
| **Smishing** | SMS phishing — fake text messages |
| **Email spoofing** | Forging the sender address to appear legitimate |
| **Malicious attachments** | Email attachments containing malware |
| **Malicious links** | Links that redirect to fake or infected websites |

### Identifying phishing emails:
- Urgent or threatening language ("Your account will be suspended!")
- Sender email address doesn't match claimed organisation
- Hover over links — URL doesn't match display text
- Generic greeting ("Dear Customer" instead of your name)
- Requests for passwords, PINs, or banking details
- Grammar and spelling errors

## Protection Against Email Threats

| Measure | How it helps |
|---|---|
| **Spam filter** | Automatically moves suspicious email to junk/spam |
| **Antivirus** | Scans attachments before opening |
| **2FA on email accounts** | Even if password is stolen, account can't be accessed |
| **Don't click unknown links** | Visit websites directly by typing the URL |
| **Verify requests** | Phone the organisation directly to confirm requests |
| **Keep software updated** | Patches vulnerabilities exploited by malware |
| **Check sender address** | Look at actual email address, not just display name |

## Online Communication and Privacy

| Issue | Consideration |
|---|---|
| **Digital footprint** | All online activity leaves traces — emails, posts, chats |
| **Cloud storage of messages** | Emails on cloud servers can be accessed by provider |
| **End-to-end encryption** | WhatsApp, Signal — only sender/receiver can read |
| **Data retention** | Many services keep your messages for years |

## Exam Focus

> [!IMPORTANT] High-Frequency Questions
>
> 1. **"Explain the difference between POP3 and IMAP"** — POP3 downloads email to one device and deletes from server; IMAP keeps email on the server and syncs across multiple devices
>
> 2. **"What is VoIP? Give ONE advantage and ONE disadvantage"** — Voice over IP — converts voice to data packets sent over internet; advantage: cheap/free international calls; disadvantage: quality depends on internet connection
>
> 3. **"Give FOUR rules of netiquette"** — don't type in all caps; use clear subject lines; don't forward without permission; reply promptly; be concise
>
> 4. **"How can you identify a phishing email?"** — urgent language; sender address doesn't match; hover over links to see real URL; generic greeting; requests for passwords/banking details; spelling errors
>
> 5. **"What is the difference between phishing and spear phishing?"** — phishing is a mass email to many people; spear phishing is targeted at a specific individual and is more convincing as it uses personal details
