# Solution Development — Theory

This section covers the theoretical underpinnings of programming and software development as assessed in Paper 2. While you will write actual Delphi code and SQL in Paper 1, Paper 2 tests your ability to describe, explain and trace algorithms, OOP principles and software development concepts without a computer.

## What's in this section

| Page | What you'll learn |
|---|---|
| [Programming Concepts](./programming-concepts) | Variables, data types, constants, operators, control structures — the theory side |
| [Algorithms (Theory)](./algorithms-theory) | What an algorithm is, pseudocode, flowcharts, tracing algorithms by hand |
| [OOP Principles](./oop-principles) | Classes, objects, encapsulation, inheritance, polymorphism, abstraction |
| [Software Engineering](./software-engineering) | SDLC, problem analysis, testing strategies, documentation, maintenance |

## CAPS alignment

Solution Development theory is examined in **Paper 2** across all grades.

- **Grade 10** — Basic algorithm concepts, pseudocode and flowcharts, simple program logic.
- **Grade 11** — Algorithm tracing, OOP introduction, software development life cycle.
- **Grade 12** — OOP principles in depth, advanced algorithm analysis, software engineering concepts.

> **Tip:** The practical counterpart to this theory is in the [Practical section](../../practical/) — specifically [Algorithms](../../practical/algorithms/) and [Delphi](../../practical/delphi/).
